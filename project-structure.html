design-mind-mcp (root)
MCP server · UI genome · pattern library · candidate API
Top-level modules (siblings)
⚙️
server/MCP Core
stdio / SSE MCP server — exposes the three tools to Claude Code
📄
src/index.js
Entry point — registers tools, handles transport (stdio or SSE)
📄
src/contextAssembler.js
Builds the consult_before_build response — surface + patterns + rules + ontology + safety
📄
src/knowledge.js
Loads and caches genome/ontology/pattern data from disk
📄
src/embedder.js
Generates embeddings for semantic search over patterns & rules
📄
src/vectorIndex.js · vectorstore.js
In-memory + optional Qdrant vector index for similarity search
📄
src/seed.js · seed-vectors.js · seed-qdrant.js
One-time scripts to (re)index all patterns & genome content
📋
mcp-config.json
Declares the three MCP tools with JSON Schema
🌐
api/Candidate API
Standalone HTTP server for candidate submission (report_pattern target)
📄
src/index.js
Express API — POST /candidates, deduplication logic, Slack webhook notification, frequency counter & ratification flag
📋
package.json
API-specific dependencies (Express, etc.)
🧬
genome/Design DNA
Core design rules, aesthetic taste, and high-level principles
📄
principles.md
High-level design philosophy and decision-making framework
📄
taste.md
Aesthetic preferences — what "good" looks and feels like
📁
rules/
5 atomic rule files: styling-tokens · data-density · destructive-actions · navigation-patterns · shell-layout + _index.json
📖
ontology/Vocabulary
Canonical names for entities, actions, states, copy voice, and identifier labeling — shared language for all surfaces
📄
entities.yaml
Domain entities (Patient, CareGap, Task, Provider…)
📄
actions.yaml
Canonical verbs (dismiss, schedule, escalate…)
📄
states.yaml
Lifecycle states (open, closed, in-progress, overdue…)
📄
copy-voice.md
Tone, phrasing rules, copy patterns, and forbidden terminology for UI text
🧩
patterns/UI Patterns
Ratified UI patterns — each in its own sub-folder with a spec and optional component
📁
ActionableRow/
Composition — list row with primary action; variant="row" or "card"
📁
StatusBadge/
Decision — any entity status display
📁
ClinicalAlertBanner/
Decision — alert requiring immediate acknowledgment
📁
PatientContextHeader/
Composition — patient identity header for any patient-scoped surface
📁
StatCard/
Decision — summary metric or count
📁
PatientRow/
Composition — patient in a population list
📁
SectionHeader · OutreachLogRow · SdohAssessmentTab · ChatQuickActionChip · InlinePatientCard/
Additional ratified patterns
📁
_candidates/
8 pending YAML proposals awaiting ratification (e.g. IntakeForm, MobileChatDrawer, AssessmentInstrumentList)
Each pattern folder contains: meta.yaml (spec) + optional component.tsx

surfaces/Full Artifacts
Surface specs — encode intent, omissions, ordering, actions, and hard "never" rules for full-page workflows
📄
Today.surface.yaml
Coordinator daily dashboard
📄
Worklist.surface.yaml
Coordinator patient worklist
📄
PatientDetail.surface.yaml
Patient detail — coordinator + clinician
📄
OutreachLog.surface.yaml
Outreach history log
📄
IntakeForm.surface.yaml
Patient intake workflow
📄
ClinicalAssessment.surface.yaml
Clinical assessment form — coordinator + clinician
🛡️
safety/Constraints
Hard constraints that can never be violated, plus a severity schema
📄
hard-constraints.md
13 rules across 5 sections — severity colors (token-based), alert dismissal, patient identity, confirmation & destructive actions, data display, and interactive states. Injected into every build consultation. Immutable without explicit clinical leadership approval.
📄
severity-schema.yaml
Defines severity levels for rule violations (error / warning / info)
🤖
agents/AI Agents
System prompts for specialized AI agents used internally
📄
mind/system-prompt.md
Primary design-generation agent prompt
📄
critic/system-prompt.md
Review/critique agent prompt — drives review_output
🔌
adapters/Integrations
Pluggable LLM adapter layer — swap model backends without touching server logic
📄
base.adapter.interface.ts
TypeScript interface all adapters must implement
📄
claude.adapter.ts
Concrete adapter for Anthropic Claude API
💾
memory/Persistence
Episodic build log and vector DB config for session memory
📄
episodic-log.jsonl
Append-only log of past build consultations and decisions
📄
vector-db.config.yaml
Qdrant connection config for production vector search

tools/Schemas

📄
review_output.schema.yaml
Input schema — generated component code for review
📄
report_pattern.schema.yaml
Input schema — new pattern candidate submission payload
📜
scripts/Utilities
Dev-time scripts for maintenance tasks

generate-context.js
Assembles the project-context-full.md snapshot for Claude
📄

Reviews _candidates/ and surfaces those ready for ratification
